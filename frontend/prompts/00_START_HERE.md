# Start Here

Use this prompt when beginning a frontend AI work session.

## Prompt

Read these files first:

- `AGENTS.md`
- `frontend/RULES.md`
- `frontend/docs/AI_RUNBOOK.md`
- `frontend/docs/framework/PROJECT_CONTEXT.md`
- `frontend/docs/framework/UI_WORKFLOW.md`
- `frontend/docs/framework/DESIGN_RULES.md`
- `frontend/docs/framework/DESIGN_SYSTEM.md`
- `frontend/docs/framework/SCREEN_SPECS.md`
- `frontend/docs/framework/AI_CONSTRAINTS.md`

Then:

1. summarize the current frontend context
2. inspect the relevant routes and reusable components
3. identify the smallest safe implementation approach
4. update generated docs if the task is multi-screen or design-heavy
5. implement code that follows `frontend/RULES.md`
6. run validation commands when feasible
7. report changes, risks, and any doc updates needed
